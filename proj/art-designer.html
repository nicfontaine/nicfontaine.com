Mursion Art Designer

←   Back to Projects

Why

Initially, Mursion's 3D art configurations (art-bundles) were mass-deployed as extensive presets, per industry and setting. Example: Corporate Conference Room with almost 100 combinations of corporate perspectives, 1-5 avatars per scene, in business attire. And repeat, for every industry/setting type, or custom client artwork.

Naturally, this ends up being restrictive and repetitive and requires manual updates from the Art team - the only users with the tools and knowledge to create new configurations. As the Operation Team designs scenarios and reviews artwork with clients prior to requesting updates - they should have the power to launch new art work as-needed, from a rolling release library.

Objectives

  • Launch new art configurations faster, to reduce contract timelines
  • Pair scenario design with art design, and offload operational work from artists
  • Create a database of art assets, with meta information for filtering, customization, and configuration
  • Launch a visual library of Mursion artwork, that can be updated quickly without any software deployments
  • Create the baseline code and UX to eventually enable clients and their learners to view assets and customize their own scenarios

Components

1. Art Designer Front-End

Source & display 3D artwork, create visual builder, add filtering logic, connect submission forms to live pipeline

2. Designer Service Backend

APIs to query 3D artwork assets, modify art-project configurations, trigger build pipelines, and create tracking tickets

3. Art Library

Local application to import 3D assets from Unity3D, define meta rules & interdependencies, configure default settings, and upload images

Challenges

  • Assets can't always be freely combined, and need rules in place
    • Each environment has a number of camera perspectives, each with a separate set of seats
    • Each avatar support a number of animations, optomized for body shape, setting context, gender, etc.
    • Avatars and outfits may be gendered, uniquely sized, customized per client, etc.
  • No listing of any of this information existed previously, other than in spreadsheets and documentation
  • How to represent a Unity assetbundle in a web application, with custom definitions
  • How to keep the library up-to-date with fixes, improvements, and releases
  • How to import, edit, and submit from the existing library of art-projects (configurations)
  • How to distill all of this information to Operational users, without art-pipeline knowledge
